# Terraform Tool

Opsy has a dedicated Terraform tool with plan-before-apply workflow and automatic state backups.

## Setup

1. Install [Terraform](https://www.terraform.io/downloads)
2. Navigate to a directory with `.tf` files

## Usage

Run `opsy` to start interactive mode, then ask:

```
> initialize terraform
> plan changes for this terraform project
> apply the terraform plan
```

## Plan-Before-Apply Workflow

Opsy enforces a safe workflow:

1. **Plan** - Opsy auto-saves plan files to `~/.local/share/opsy/terraform/<project>/plans/`
2. **Apply** - Must reference a plan file from step 1

```
You: "apply terraform changes"
Opsy: First, let me create a plan...
      [runs terraform plan -out=<auto-generated-path>]
      Plan saved. Ready to apply?
You: "yes"
Opsy: [runs terraform apply <plan-file>]
```

## State Backups

Before any state operation (`terraform state mv`, `terraform import`, etc.), opsy automatically backs up your state via `terraform state pull`.

Backups are saved to: `~/.local/share/opsy/terraform/<project>/backups/`

## Command Types

| Type | Examples | Permission |
|------|----------|------------|
| read | `show`, `output`, `state list` | Auto-approve |
| plan | `plan` | Ask |
| apply | `apply` | Ask + requires plan |
| destroy | `destroy` | Ask + requires plan |
| state | `state mv`, `import` | Ask + auto-backup |

## Configuration

```jsonc
{
  "permission": {
    "terraform": {
      "*": "ask"
    }
  }
}
```
